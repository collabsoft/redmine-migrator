1. Intro

This is the documentation for the Redmine to Taiga migrator. You can find the source code of this project in the Taigaio Github account.

2. Download & Execution

First of all you need to download the Redmine to Taiga migrator. Please download the lastest version of the migrator. As a dependency you only need Java in your system.

You need JDK 7 to execute the importer

Now just execute:

java -jar redmine-migrator-<version>.jar
Figure 1. Main window

3. Configuration

3.1. Redmine

To use the Redmine Rest API you need to connect to your Redmine instance as an Administrator user and go to Administration → Settings → Authentication and enable the option Enable REST web service.

Figure 2. Redmine settings
Unfortunately the migration process needs to be done with the Redmine administrator user because the migrator needs to get the email of the users to assign them the issues, comments,…​ and it is only expose for administrators.

Go to your Redmine profile page and copy the API access key because it is the credentials you will need to login into Redmine Rest API.

Figure 3. Redmine user settings

Then you can go to the migrator Project → Settings and fill the form with your Redmine info:

Figure 4. Redmine settings

  • Address: The url where Redmine instance is available

  • ApiKey: The Api access key you’ve already copied from Redmine

  • Timeout: If your Redmine instance is in a very slow network you may need to change this setting to a higher value. This value should be set in milliseconds.

3.2. Taiga

Go to Project → Settings and fill the form with your Redmine and Taiga credentials

Figure 5. Taiga settings
Please note that you can use any Taiga instance to migrate your projects. For example, if you want to migrate to the hosted Taiga environment you need to use https://api.taiga.io as Taiga address.

4. Migration

If your settings are correct you can click on Redmine button or Ctrl-r to load all your Redmine projects.

Figure 6. Redmine projects

Now you can select one or more projects and click on the migrate button. Please note the migrator allows multiple selection interval using CTRL key.

Figure 7. Confirm project migration
Please open the log before the migration starts to check the progress.
Figure 8. Migrating projects
Figure 9. Project migrated

Open Taiga projects in Projects → Taiga Projects and you can check that the project has been migrated.

Figure 10. Taiga projects with the migrated one

Please check the log for errors and warnings during the migration process.

Sometimes could be useful to export the log. In the Log window there are two buttons, one to clear the log and another one to export what is in the log so far.

Figure 11. Export logs

By default the log will be exported to the user home directory with a name like 'taiga_migrator_[timestamp].log`.

5. Postmigration tasks

With your projects migrated into Taiga you can now connect to your Taiga instance and adjust the project configuration.

Figure 12. Your project has been migrated

Please select the project and go to the admin menu. Now you can do a fine tunning of all the options in your project:

  • Custom attributes: You can rearrange and change the color of the issue statuses, types, priorities and severities.

  • Roles & Permissions: Create new roles and change the permissions of the current roles.

  • Manage members: Change the profile of the users and invite new users to your Taiga project.

  • Project Profile: Enable or disable the Taiga modules you want to use and also set the default values for issue priority, severities,…​

6. Limitations and known issues

You need to keep in mind that the migration process is not perfect because Redmine and Taiga features don’t fully overlap each other.

You also need to check that your Redmine version expose in the REST API all the objects that are going to be migrated. For example, until Redmine 2.2 the Wikipages aren’t exposed by the REST Api.

In the following sections we explain the migration decisions for every element in Redmine.

6.1. Projects

The projects are migrated with the same name and description as in Redmine. A new slug is created to avoid name colisions with other users. You can modify all these values in Admin → Project profile → Project details.

6.2. Users

We made the decision of do not migrate users. The users must previously exist in Taiga before the migration with the same email as in Redmine. During the migration process we use the email of the users to asign them as the owner of an issue, the comments,…​

6.3. Issues

All the issues of the projects are migrated to Taiga and the Redmine trackers, priorities and status are also migrated. For every issue besides the basic data we migrate the comments, attachments and assigned user.

6.4. Wiki

Wikis are one of those cases where the migration is not perfect. Redmine supports different markup languages such as textile or reStructuredText. Taiga wikis are written in Markdown so you might need to fix some formats after the migration. Besides the text of the wikipages, all the attachments are also migrated to the wikipages.

Another important thing is that in Taiga the main wiki page is called Home and in Redmine, by default it is Wiki. Even so it’s possible to change the default wiki page in Redmine. As Redmine API doesn’t expose the name of the default wiki page we try to figure it out which is the default page. We try with home or wiki and if none of these pages exists in Redmine, we set the oldest page as the new home page.

6.5. Files

Currently Redmine does not expose the files in the REST API, so it is not possible to migrate them.

6.6. Documents

Currently Redmine does not expose the documents in the REST API, so it is not possible to migrate them.

6.7. Sprints, User stories and Tasks

All the issues in Redmine are migrated to issues in Taiga, so after the migration the backlog will be empty. We use in Redmine the backlogs plugin for scrum. For the next major release of the migrator we’re going to migrate the sprints, user stories and tasks of the Redmine Backlogs plugin to Sprints, User stories and Tasks in Taiga.
